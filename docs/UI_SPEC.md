# 斩龙 · UI 功能说明文档

> 供 UI 设计参考，包含所有界面、组件、交互和状态说明。

---

## 一、产品定位

**游戏语音宏工具** —— 用中文语音指令触发键盘操作，面向 PC 游戏玩家。  
目标平台：Windows（PySide6 桌面应用，打包为 EXE）。

**核心使用流程**：
1. 用户配置语音宏（低频操作）
2. 点击「开始监听」
3. 切到游戏全屏，说话触发按键

---

## 二、窗口一览

| 窗口 | 类型 | 说明 |
|---|---|---|
| 主窗口 | 常驻 | 宏管理 + 监听控制 |
| 日志窗口 | 子窗口 | 开启监听时自动弹出，显示识别日志 |
| 关于窗口 | 子窗口 | 首次启动弹出，免责声明 + 打赏 + 交流群 |

## 三、主窗口

### 3.1 顶栏

| 元素 | 说明 |
|---|---|
| 应用图标 + 标题 | 「🐉 斩龙」，左对齐 |
| 监听按钮 | **最核心的操作**，右对齐，醒目 |

**监听按钮两个状态**：

| 状态 | 文案 | 颜色建议 |
|---|---|---|
| 空闲 | 🔊 开启监听 | 蓝色/主色调 |
| 监听中 | 🔇 监听中... | 红色/警告色，不可再次点击 |

---

### 3.2 宏列表区（主体）

展示所有已配置的语音宏。每条宏需要展示 **3 个字段**：

| 字段 | 说明 | 示例 |
|---|---|---|
| **名称** | 用户自定义的标识 | 踩、复制、点灯 |
| **语音** | 拼音序列（由录音识别生成） | c ái |
| **按键** | 触发的键盘操作 | w、(ctrl, c)、double_click:space |

**每条宏的可用操作**：
- ✏️ **编辑** — 修改语音和按键（名称不可改，改名需删除重建）
- 🗑️ **删除** — 需二次确认
- 每条可直接内联编辑，也可以用独立的编辑区/弹窗，设计上自由发挥

**列表底部/其他位置**：
- ＋ **添加新宏** — 打开新建表单

---

### 3.3 新建/编辑表单

新建和编辑共用同一套表单，包含 3 个输入项：

| 输入项 | 类型 | 说明 |
|---|---|---|
| **操作命名** | 文本输入框 | 自由命名，新建时必填；编辑时只读 |
| **录入按键** | 文本输入框 | 按键之间空格隔开，组合键用括号 `(ctrl,c)` |
| **录入语音** | 文本输入框 + 🎤话筒按钮 | 长按话筒录音 → 松开后显示识别结果（拼音） |

**话筒按钮交互**：

| 动作 | UI 变化 |
|---|---|
| 按下 | 按钮变红/高亮，输入框显示「🎤 录音中...」 |
| 松开 | 输入框显示「⏳ 识别中...」→ 最终显示拼音结果 |
| 识别失败 | 显示错误提示 |

**表单按钮**：
- 「保存」/「添加」— 主操作
- 「取消」— 关闭/收起

---

### 3.4 按键输入格式说明

用户在「录入按键」框里输入的语法（可以在 placeholder 或 tooltip 里提示）：

| 写法 | 含义 | 示例 |
|---|---|---|
| `a` | 单个按键 | `w`、`2`、`f11` |
| `a b c` | 顺序按多个键 | 按 a → 按 b → 按 c |
| `(ctrl,c)` | 组合键 | Ctrl+C |
| `double_click:space` | 双击某个键 | 双击空格 |

## 四、日志窗口

**触发时机**：点击「开始监听」后自动弹出。

| 元素 | 说明 |
|---|---|
| 日志区域 | 只读文本区，实时追加日志，自动滚动到底部 |
| 清空按钮 | 清空当前日志 |

**日志内容示例**：
```
🟢 语音监听已启动
🎯 触发技能: 踩
🎯 触发技能: 复制
🔴 语音监听已停止
```

**关闭行为**：关闭日志窗口 = 自动停止监听。

## 六、数据模型

底层只有两份配置文件：

### skills.json — 宏映射
```json
{
    "踩": ["w"],
    "复制": [["ctrl", "c"]],
    "双击空格": ["a", ["double_click", "space"], "c"]
}
```

### keywords_invoker.txt — 语音关键词
```
c ái @踩 :0.35
ch ái @踩_v1 :0.40
f ù zh ì @复制 :0.25
```
> 方言变体（_v1, _v2...）由系统自动生成，用户不感知。

## 七、整体状态流

```
                    ┌──────────────────┐
                    │   应用启动        │
                    └────────┬─────────┘
                             │
                    ┌────────▼─────────┐
                    │  加载配置文件      │
                    │  显示主窗口       │
                    │  弹出关于窗口      │
                    └────────┬─────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        管理宏配置      点击监听按钮     关闭窗口→退出
        (增删改)            │
              │       ┌─────▼──────┐
              │       │ 监听状态    │
              │       │ 弹出日志窗口│
              │       │ 按钮变红    │
              │       └─────┬──────┘
              │             │
              │       语音触发 → 日志显示
              │             │
              │       关闭日志/再次点击
              │       → 停止监听
              │       → 按钮恢复蓝色
              └─────────────┘
```

## 八、设计建议（非强制）

- 目标用户是游戏玩家，风格可以偏**游戏 UI / 暗色**
- 主窗口建议尺寸约 **800-960 × 600-720**
- 宏列表是核心区域，应该占大部分空间
- 监听按钮是最高频操作，需要**最醒目**
- 编辑操作是低频的，可以适当收敛（弹窗/侧边栏/内联都行）
- 需要兼顾 Windows 10/11 的系统渲染
